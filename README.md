# Zepto_sql_analysis
This is a complete, real-world data analyst portfolio project based on an e-commerce inventory dataset scraped from Zepto — one of India’s fastest-growing quick-commerce startups. This project simulates real analyst workflows, from raw data exploration to business-focused data analysis. In this we have used postgresql.


# 🛒 Zepto Data Analysis Project (SQL + Python)

## 📌 Overview

This project focuses on analyzing Zepto’s product and inventory dataset using **Python and SQL** to uncover business insights. It demonstrates an end-to-end data analytics workflow, including data loading, exploratory data analysis (EDA), data cleaning, and SQL-based analysis.

The objective is to derive insights related to pricing, discounts, stock availability, and product categories to support better business decision-making.

---

## 📂 Dataset

* The dataset contains product-level information from Zepto, including:

  * Product name and category
  * MRP (Maximum Retail Price)
  * Discount percentage and selling price
  * Available quantity and stock status
  * Product weight

* The dataset includes real-world challenges such as missing values, inconsistent pricing, and out-of-stock items.

---

## 🛠️ Tools & Technologies

* **Python (Pandas, NumPy, Matplotlib/Seaborn)** – Data cleaning and EDA
* **SQL (PostgreSQL / MySQL / SQL Server)** – Data querying and analysis
* **Jupyter Notebook / VS Code** – Development environment

---

## 🔄 Project Steps

### 1. Data Loading

* Imported the Zepto dataset into Python using Pandas
* Checked data structure, column types, and summary statistics

### 2. Exploratory Data Analysis (EDA)

* Analyzed product distribution across categories
* Examined pricing, discounts, and stock availability
* Identified trends such as:

  * High discount categories
  * Frequently out-of-stock products
  * Price variations across categories

### 3. Data Cleaning

* Handled missing and null values
* Removed duplicates and corrected inconsistencies
* Standardized numerical columns (price, discount, quantity)
* Ensured accurate stock status representation

### 4. SQL Analysis

* Created structured tables in SQL database
* Wrote queries to extract insights, such as:

  * Top discounted products
  * Category-wise average pricing
  * Stock availability analysis
  * Revenue estimation based on selling price and quantity

---

## 📊 Key SQL Insights

* Categories with the highest average discounts
* Products with maximum price difference between MRP and selling price
* Out-of-stock trends across categories
* High-value products contributing to potential revenue
